## 09 · 后端框架选型

> **最终决定**：基于 [tiangolo/full-stack-fastapi-template](https://github.com/fastapi/full-stack-fastapi-template)（本地副本：`/data/gzl/project/lili-full-stack-master`）二次开发。
> 后端：**FastAPI + SQLModel + PostgreSQL + Alembic + JWT**
> 管理后台：**React 19 + Vite + TanStack + shadcn/ui**（模板自带）
> 部署：**Docker Compose + Traefik（自动 HTTPS）**

### 1. 决策过程

| 阶段 | 备选 | 结论 |
| --- | --- | --- |
| 一开始 | 微信云开发 cloudfunctions | 否决：复杂业务受限、调试差、绑死腾讯生态 |
| 第二轮 | NestJS / Django / FastAPI 自建 | Django Admin 强但定制差；NestJS admin 要从零做 |
| **最终** | 复用 full-stack-fastapi-template | ✅ 后端 + 后台前端 + Docker + CI 全有，省 1-2 周 |

### 4. User 模型扩展（关键设计）

用同一张表既给小程序用户又给后台管理员：

```python
class User(UserBase, table=True):
    id: uuid.UUID = Field(default_factory=uuid.uuid4, primary_key=True)
    # 模板原有
    email: EmailStr | None = Field(default=None, unique=True, index=True)  # 改成可空
    hashed_password: str | None = Field(default=None)  # 同上
    is_active: bool = True
    is_superuser: bool = False
    full_name: str | None = None
    # 新增（小程序用户字段）
    openid: str | None = Field(default=None, unique=True, index=True)
    unionid: str | None = None
    xy_code: str | None = Field(default=None, unique=True, index=True)
    unlock_balance: int = 3
    notify_enabled: bool = False
    verified: str = "none"  # none/pending/passed/rejected
    last_active_at: datetime | None = None
    # 关联
    profile: "Profile" = Relationship(back_populates="user")
    criteria: "Criteria" = Relationship(back_populates="user")
```

**鉴权分流**：
- `/login/access-token`（已有）：邮箱+密码 → JWT，给后台用
- `/wechat/login`（新增）：code → openid → JWT，给小程序用
- 两个 token 同一种格式，`get_current_user` 都能解；`get_current_active_superuser` 自动卡住非 admin

#### 分仓的理由

1. **小程序随时拉走**：`qianchenhunlian/` 几 MB，复制到 Windows 用微信开发者工具直接打开
2. **服务端整体迁移**：`tar czf server.tar.gz qianchenhunlian_server/` → 任意服务器 rsync → `docker compose up -d` 包括数据库数据一并复活
3. **数据库文件 bind mount 在仓库目录**：`data/postgres/` 直接是 PG 数据目录，备份 = 备份目录，迁移 = 复制目录
4. **不破坏 admin 与 backend 的紧耦合**：它们共享 docker-compose、.env、共用 OpenAPI；放在同一个 server 仓内最合理

#### 分仓的代价（可控）

- **枚举要手动同步**：03/04 文档是 single source of truth，改后端枚举必须同步小程序 `constants/enums.js`
- **跨仓搜索**：VSCode 用 multi-root workspace（同一窗口加两个 root）即可
- **CI 配两份**：每个仓一套 GitHub Actions
- **OpenAPI client**：admin 在 server 仓内正常生成；小程序不依赖 TS 类型

### 7. 鉴权流程（小程序版）

```
小程序                          FastAPI
  │ wx.login() → code            │
  ├──POST /wechat/login─────────▶│
  │  { code }                    │
  │                              │  调微信 jscode2session API
  │                              │  拿 openid (+ unionid)
  │                              │  upsert User（email 留空）
  │                              │  生成寻缘号
  │                              │  签 JWT（同模板原有逻辑）
  │◀── { access_token, user } ───┤
  │                              │
  │ Header: Authorization: Bearer│
  ├──任意业务接口─────────────────▶│
  │                              │  Depends(get_current_user) 解 JWT
  │                              │  路由内业务逻辑
  │◀── 返回 ─────────────────────┤
```

后台管理员鉴权完全沿用模板原有 `/login/access-token` 邮箱密码登录，不动。

### 10. 风险与已知坑

- 模板 User 的 `email` 现在是必填且唯一。改成可空 + 唯一非空索引（`email IS NOT NULL`）。Alembic 迁移要小心，最好新项目第一次 init 时直接改完。
- 模板 Items 表用 `owner_id` 关联 user，删 user 级联删 item。我们的 Profile 也要 `ondelete="CASCADE"`。
- FastAPI 没自带定时任务。**APScheduler** 直接嵌应用够用；任务多了再上 Celery + Redis broker。
- 模板的 frontend 用 TanStack Router 文件路由，加新页面要 `pnpm generate-routes`（脚本里有）；不能直接手写 routes 文件。
- 小程序合法域名要 HTTPS + 备案过的域名，dev 阶段先用云开发的免费域名兜底，或用 trycloudflare/ngrok（可能被微信拒）。
- 模板用 `passlib + bcrypt`，密码登录无忧；微信登录的 `session_key` 不写 user 表（敏感、可滚动），需要时缓存到 Redis 短期保存。

### 12. 备选

如果出于任何原因要换：
- **保 Python 但要换 admin 重的方案** → Django + DRF + Django Admin
