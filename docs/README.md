### 开发顺序建议
1. 两位开发一起读完 01–04，明确边界。
2. 后端先把 03 的 DB 初始化脚本与 04 的接口契约打通（哪怕先返回 mock 数据），前端立即切换到调真接口、淘汰 `data.js` mock。
3. 按 06 的里程碑推进，每个里程碑收尾做一次联调 + demo。
