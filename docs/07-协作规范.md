## 07 · 协作规范

### 1. Git 分支模型
- `main`：始终可上线，只能通过 PR 合入。
- `develop`（可选）：如果多人并行较多可启用，否则省略。
- 功能分支命名：`feat/<模块>-<简述>`，例：`feat/profile-save-api`、`feat/detail-unlock-ui`
- 修 bug：`fix/<简述>`，例：`fix/pref-row-alignment`

### 2. Commit 规范
用常见的 Conventional Commits 风格，小步提交：
- `feat(cf-profile): 新增 saveProfile 接口`
- `feat(mp-detail): 对接 match.getProfile`
- `fix(mp-my-profile): 择偶要求 label 对齐`
- `docs: 更新 04-云函数接口 错误码说明`
- `chore: 删除示例 example 页`
- `refactor(mp): 抽取 services/profile.js`

规则：
- 一个 commit 只做一件事，避免"改了资料页顺便改了登录"的混合提交
- 提交信息中英文均可，保持团队内一致
- 禁用 `--no-verify` 跳过钩子

### 3. PR 规范
- 标题与 commit 规范一致
- 描述必须包含：改动范围 / 测试方式 / 需要哪边联调
- 模板：
  ```
  ### 改动
  - 新增 XXX
  - 修正 YYY

  ### 测试
  - [ ] 已在开发者工具本地跑通
  - [ ] 联调: 需要 @对方 在 devEnv-xxx 一起看

  ### 关联
  docs/04-云函数接口.md#X.Y
  ```
- 另一人 approve 后合并；紧急情况自评并补上 follow-up。

### 4. 代码风格
小程序端：
- 2 空格缩进，单引号
- 页面 js 中业务相关数据放 `data`，纯视图辅助用 computed/临时变量
- 所有调云函数通过 `services/`，页面里不 `wx.cloud.callFunction` 直接裸调
- 常量（学历/收入/婚况等枚举）放 `constants/`，前后端必须保持一致，后续考虑用 ts 或 json 共享
- wxss 颜色集中到 `app.wxss` 的 CSS 变量（后续迭代）

云函数端：
- 每个云函数 `index.js` 以 `event.action` 路由，子处理器放 `./handlers/<name>.js`
- 入参必须白名单 + 类型校验
- 错误统一抛 `throw { code, msg }`，在 main 里 catch 成标准响应
- 任何 db 写入后返回的数据要是"真实最新"版本，避免前端再拉一次

### 5. 字段与枚举一致性
- 学历、收入、婚况、婚房等枚举：后端以 `profile.saveProfile` 的白名单为准；前端常量必须和后端同步
- 日期字段：后端一律 `db.serverDate()`，前端格式化时用 `utils/date.js`（待建）

### 6. 联调约定
- 后端接口 ready 前先提供 mock：在 `index.js` 里对 action 硬编码返回示例数据，前端可以先接起来
- 前端在页面显示"接口 mock 中"标记（dev 环境开关）
- 联调时两人一屏幕，一次对一个 action

### 7. 文档更新
- 任何改接口的 PR 必须同时改 `docs/04-云函数接口.md`
- 任何改数据库 schema 的 PR 必须同时改 `docs/03-数据库设计.md`
- `docs/08-待确认事项.md` 有新增 / 关闭时，带上日期和决策人

### 8. 版本与发布
- 开发环境：每次 PR 合入后由 PR 提交者部署到公用 devEnv
- 预发布：MVP 联调完成后申请一个 testEnv，用来体验版
- 生产：上线前最后一次 PR 带 tag `v1.0.0`

### 9. 敏感信息
- 不要把 env id、管理员 openid、模板 id 提交到仓库
- 这类配置放 `miniprogram/envList.js`（已 gitignore 或另行约定）或云函数的 `config.json`（后端侧）
- 联系方式字段在云函数内处理，永远不要在前端日志或错误信息里打印

### 10. 代码检查清单（Code Review 前自检）
- [ ] 新增字段是否同步到 03 文档？
- [ ] 新增 action 是否在 04 文档？
- [ ] 有没有打印 openid 到生产日志？
- [ ] 新增云函数是否做了入参校验？
- [ ] 前端 loading / 错误态 / 空态是否处理？
- [ ] `pages/example/`、`quickstartFunctions` 有没有被新代码引用？（应全部清掉）
