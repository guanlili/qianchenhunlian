## 04 · 后端接口契约（原"云函数接口"）

> 📌 项目已切换为自建 NestJS 后端，本文档原"云函数 + action"模型已转为 RESTful HTTP API，**接口语义保持不变**。具体 endpoint 列表见 [09-后端框架选型.md](09-后端框架选型.md) 的第 10 节。
> 下方仍以 action 名描述业务语义（便于前端理解每个调用要做啥），实际调用时按 09 的 endpoint 表对应即可。

### 统一约定
- 调用方式：`wx.request({ url: BASE_URL + '<endpoint>', method, header: { Authorization: 'Bearer <jwt>' }, data })`
- 老的 `wx.cloud.callFunction({ name, data: { action, ...payload } })` 已废弃
- 返回结构（成功/失败一律外层 resolve，错误在 body 里）：
  ```json
  { "code": 0, "data": { ... }, "msg": "ok" }
  ```
- 错误码：
  | code | 含义 |
  | --- | --- |
  | 0 | 成功 |
  | 1001 | 参数错误 |
  | 1002 | 未登录 / 用户不存在 |
  | 1003 | 权限不足 |
  | 1004 | 资源不存在 |
  | 2001 | 解锁余额不足 |
  | 2002 | 重复操作 |
  | 2003 | 资料未通过审核 |
  | 9000 | 服务端异常 |
- 所有写操作服务端校验：`openid = wxContext.OPENID`，**不信任前端传入的 openid**。
- 时间全部用 `db.serverDate()`。

### 云函数 1：`user` · 用户与账号

#### 1.1 `user.login`
首次调用会注册用户并发放寻缘号；之后的调用幂等返回。
- 入参：`{ action: "login" }`
- 出参：
  ```json
  {
    "code": 0,
    "data": {
      "openid": "...",
      "xyCode": "53652255",
      "unlockBalance": 3,
      "notifyEnabled": false,
      "verified": "none",
      "hasProfile": false,
      "hasCriteria": false
    }
  }
  ```
- 小程序 onLaunch / 首页 onShow 调用；用于判定首屏要不要跳"填资料"。

#### 1.2 `user.getProfileSummary`
返回我的资料 + 择偶要求 摘要，供"相亲资料"页使用。
- 入参：`{ action: "getProfileSummary" }`
- 出参：`{ code, data: { user, profile, criteria, stats } }`

#### 1.3 `user.updateNotify`
- 入参：`{ action: "updateNotify", enabled: true }`
- 作用：更新 `users.notifyEnabled`，记录订阅消息 accept 时间。

#### 1.4 `user.reportFollow`
用户完成关注公众号后客户端上报。
- 入参：`{ action: "reportFollow" }`

### 云函数 2：`profile` · 我的资料 + 择偶要求

#### 2.1 `profile.get`
- 入参：`{ action: "get" }`
- 出参：我自己的 profile + criteria 完整字段。

#### 2.2 `profile.saveProfile`
- 入参：`{ action: "saveProfile", data: { gender, year, height, edu, income, marriage, origin, location, hometown, job, hasHouse, hasCar, bodyType, desc, photos, relation, contact } }`
- 逻辑：
  - 如无记录则 insert，否则 update。
  - 所有字段做白名单过滤。
  - 重新计算 `progress`。
  - 若资料已被 approved，修改后把状态回退到 `pending`。
- 出参：最新 profile。

#### 2.3 `profile.saveCriteria`
- 入参：`{ action: "saveCriteria", data: { year, height, income, edu, marriage, house, origin, location, note } }`
- 同上。

#### 2.4 `profile.uploadPhotoCommit`
图片上传走 `wx.cloud.uploadFile`（客户端直传），拿到 `fileID` 后调本接口把它挂到 profile.photos。
- 入参：`{ action: "uploadPhotoCommit", fileID: "cloud://...", index: 0 }`
- 出参：最新 photos 数组。
- 后端同时调用云开发的 `checkContent`（内容安全）做图片审核（可选，一期先留钩子）。

#### 2.5 `profile.deletePhoto`
- 入参：`{ action: "deletePhoto", index: 2 }`
- 同步删除云存储里的文件。

### 云函数 3：`match` · 推荐、筛选、详情

#### 3.1 `match.getDailyList`
- 入参：`{ action: "getDailyList", tab: "today" | "new" | "missed", cursor?: string }`
  - `today`：今日推荐（读 `daily_recommendations` 当天快照）
  - `new`：看看新人（按 createdAt DESC）
  - `missed`：错过的人（前几天的推荐未点击的）
- 出参：
  ```json
  {
    "code": 0,
    "data": {
      "list": [ { "id": "...", "xyCode": "...", "gender": "...", "year": 1989, ... } ],
      "nextCursor": "xxxx"
    }
  }
  ```
- 列表项**不含**联系方式字段。

#### 3.2 `match.applyFilter`
- 入参：`{ action: "applyFilter", filter: { gender, ageRange, marriage, heightMin, eduMin, incomeMin, regions, jobs } }`
- 出参：命中列表（结构同 getDailyList）+ 存储当前筛选条件到 `users.lastFilter`（可选）。

#### 3.3 `match.getProfile`
- 入参：`{ action: "getProfile", id: "..." }`
- 逻辑：查询目标 profile → 判断调用者是否已解锁 → 决定返回字段。
- 出参：
  ```json
  {
    "code": 0,
    "data": {
      "profile": { /* 公开字段 + 敏感字段（仅已解锁时） */ },
      "unlocked": true,
      "starred": false
    }
  }
  ```
- 写一条 `views` 记录（24h 内同人只写一次）。
- 更新 `profiles.stats.hot`。

#### 3.4 `match.prevNext`
- 入参：`{ action: "prevNext", currentId, direction: "prev" | "next" }`
- 出参：上一个/下一个 profile 的 id。
- 实现上可用同一份推荐列表或按当前过滤条件的排序序号。

### 云函数 4：`favorite` · 收藏

#### 4.1 `favorite.toggle`
- 入参：`{ action: "toggle", targetOpenid: "..." }`
- 出参：`{ starred: true|false, totalFavorites: 23 }`
- 幂等：已收藏再调等于取消。

#### 4.2 `favorite.list`
- 入参：`{ action: "list", cursor? }`
- 出参：我收藏的人列表，支持分页。

#### 4.3 `favorite.viewedMe`
"看过我的人" 列表。
- 入参：`{ action: "viewedMe", cursor? }`
- 从 `views` 聚合：每个浏览人取最近一次。

---

### 云函数 5：`contact` · 解锁与联系

#### 5.1 `contact.unlock`
- 入参：`{ action: "unlock", targetOpenid: "..." }`
- 逻辑：
  - 若已解锁（`unlocks` 有记录）→ 直接返回联系方式
  - 否则检查 `unlockBalance >= 1`
  - 扣余额、写 `unlocks`、返回目标联系方式
  - 事务：余额扣减与记录写入要原子（云开发的 `db.runTransaction`）
- 出参：
  ```json
  {
    "code": 0,
    "data": {
      "contact": { "wechat": "xxx", "phone": "138xxxx" },
      "balance": 2
    }
  }
  ```
- 失败码：2001（余额不足）→ 前端提示去充值/联系客服。

#### 5.2 `contact.sendIntent`
列表"联系对方"、详情"发消息"按钮。
- 入参：`{ action: "sendIntent", targetOpenid: "..." }`
- 逻辑：
  - 写 `contact_intents`
  - 若目标 `notify_tokens` 有配额，发送订阅消息

#### 5.3 `contact.myUnlocks`
- 入参：`{ action: "myUnlocks", cursor? }`
- 出参：我解锁过的名单（用于"我解锁的联系人"入口，可选）。

### 云函数 6：`notify` · 订阅消息

#### 6.1 `notify.acceptTemplate`
客户端 `wx.requestSubscribeMessage` 成功后上报，服务端记录配额 +1。
- 入参：`{ action: "acceptTemplate", templateId: "..." }`

#### 6.2 `notify.sendDaily`
**定时触发**，非前端直调。
- 触发方式：云函数定时触发器，每日 10:00 (GMT+8)
- 逻辑：
  - 遍历 `users.notifyEnabled = true` 且 `daily_recommendations` 已生成的用户
  - 检查 `notify_tokens.remaining > 0`
  - 调 `cloud.openapi.subscribeMessage.send`
  - `remaining -= 1`

#### 6.3 `notify.buildDailyRecommendations`
**定时触发**，每日凌晨 2:00 跑。
- 为每个活跃用户（近 7 天打开过小程序）生成 `daily_recommendations` 当天文档。
- 推荐算法一期做最简单：
  1. 性别与我择偶 criteria.gender 相反
  2. 年龄/身高/学历命中 criteria 过滤
  3. 按 `profiles.updatedAt DESC` 排前 10 人
- 算法后续独立迭代，不影响前端接口。

### 云函数 7：`admin` · 管理类（后置，一期尽量在云开发控制台手动）

- `admin.initCollections`（一次性）
- `admin.approveProfile { profileId, pass, reason }`
- `admin.grantUnlock { openid, count }` 手动给指定用户加解锁次数
- 只有 `users.role = admin` 的账号能调（一期写死 openid 白名单）

---

### 接口联调顺序（建议）
1. `user.login` — 串通最小闭环
2. `profile.get / saveProfile / saveCriteria` — 我的资料页真实化
3. `match.getDailyList / getProfile` — 首页 + 详情页
4. `favorite.toggle` — 收藏
5. `contact.unlock / sendIntent` — 解锁 + 意向
6. `match.applyFilter` — 筛选
7. `notify.*` + 定时任务 — 订阅消息
