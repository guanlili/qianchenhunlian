## 05 · 页面与云函数映射

前端开发逐屏对接清单。每一屏列出：触发时机、调用的云函数、setData 的数据字段。

> 所有调用请通过 `miniprogram/services/*.js` 封装层，不要在页面里直接写 `wx.cloud.callFunction`。

---

### 屏 1：`home` · 首页每日推荐
**进入时机**：`onShow`，以及底部 tab 切回时。

| 事件 | 接口 | data 更新 |
| --- | --- | --- |
| 首次进入 | `user.login` | `userInfo`, `notifyEnabled` |
| 加载推荐 | `match.getDailyList({ tab: homeTab })` | `profiles` |
| 切换 tab（今日/新人/错过） | `match.getDailyList({ tab })` | `profiles`, `homeTab` |
| 点击"去开启"通知 | `wx.requestSubscribeMessage` → `notify.acceptTemplate` → `user.updateNotify` | `notifyEnabled` |
| 选择 relation | 暂存前端 → 下次调 `profile.saveProfile` 时带上 | `relation` |
| 点击卡片"联系对方" | `contact.sendIntent` | 显示 toast |
| 点击"查看资料" | 跳 detail 屏 `match.getProfile` | `selectedProfile` |
| 右下"填写相亲资料" | 跳 `profile-edit` 屏 | `screen='profile-edit'` |
| 顶部"筛选" | 跳 `filter` 屏 | `screen='filter'` |

**注意**：`profiles` 列表项要后端已脱敏；`id` 用 profile `_id` 或 `xyCode`，保持前后端一致。

---

### 屏 2：`detail` · 他人资料详情
**进入时机**：`openDetail(id)` 调用 `match.getProfile`。

| 事件 | 接口 | data 更新 |
| --- | --- | --- |
| 进入 | `match.getProfile({ id })` | `selectedProfile`, `detailStarred`, `detailUnlocked` |
| 点收藏 ☆ | `favorite.toggle` | `detailStarred`, `selectedProfile.likes` |
| 点"解锁对方" | `contact.unlock` | `detailUnlocked`, 弹出联系方式 |
| 点"查看联系方式"（已解锁） | 直接显示；未解锁 → `contact.unlock` | `contactOpen`, sheet 内容 |
| 发消息 | `contact.sendIntent` | toast |
| 上一个 / 下一个 | `match.prevNext` → `match.getProfile` | `selectedProfile`, `selectedProfileIndex` |
| 转发资料 | `wx.showShareMenu` → `onShareAppMessage` | - |

联系方式字段：
- 未解锁：`selectedProfile.contact` 为 `undefined`
- 已解锁：`{ wechat, phone }` 都有值

---

### 屏 3：`my-profile` · 相亲资料
**进入时机**：tab 切换到"相亲资料"。

| 事件 | 接口 | data 更新 |
| --- | --- | --- |
| 进入 | `user.getProfileSummary` | `myProfileRows`, `criteriaValues`, `profileProgress`, `stats`, `profileDesc` |
| 点"编辑资料"（我的资料） | 跳 `profile-edit` | `screen='profile-edit'` |
| 点"编辑资料"（择偶要求） | 跳 `criteria-edit` | `screen='criteria-edit'` |
| 分享到群或好友 | `onShareAppMessage` + `contact.sendIntent` 可选 | - |
| "关注公众号" 回调 | 无接口，前端 `wx.openUrl` or 引导 | - |

`stats`（点赞/人气值/看过我）来自 `profiles.stats` + `views` 聚合。

---

### 屏 4：`me` · 我的
| 事件 | 接口 | data 更新 |
| --- | --- | --- |
| 进入 | `user.getProfileSummary` | 头像/寻缘号/审核状态 |
| "看过我的人" | `favorite.viewedMe` | 列表（新页面或弹窗） |
| "我收藏的" | `favorite.list` | 列表 |
| "实名认证" | 一期占位 | - |
| "隐私设置" | 一期占位 | - |

---

### 屏 5：`profile-edit` · 资料编辑
| 事件 | 接口 | data 更新 |
| --- | --- | --- |
| 进入 | 复用 `my-profile` 拉的数据 | `profileForm`, `photos`, `profileFields` |
| 改单项（年收入/婚况/婚房） | 本地先改，保存时一次性提交 | `profileForm` |
| 上传图片 | `wx.cloud.uploadFile` → `profile.uploadPhotoCommit` | `photos` |
| 删除图片 | `profile.deletePhoto` | `photos` |
| 快速填写三问 | 提交后纯前端拼 `desc`，不单独接口 | `profileForm.desc` |
| 保存 | `profile.saveProfile` | 返回后刷新，跳回来源 |
| 暂不填写 | 跳回 | - |

进度条：保存前本地估算显示，保存后用服务端返回的 `progress`。

---

### 屏 6：`criteria-edit` · 择偶要求
对接同 profile-edit，接口换成 `profile.saveCriteria`。居住地要求的多选走 option sheet。

---

### 屏 7：`filter` · 筛选
| 事件 | 接口 | data 更新 |
| --- | --- | --- |
| 进入 | 本地选项常量 | `filterSteps` |
| 下一步 | 纯前端 | `filterStep`, `filterCurrent` |
| 完成 | `match.applyFilter` | 跳回 home，刷新 `profiles` |

筛选条件可选择是否持久化到 `users.lastFilter`（一期可放一边）。

---

### 屏 8：`quick-fill` · 三问快速生成
纯前端逻辑，不调接口。最终拼 desc 后由 `profile.saveProfile` 存库。

---

### 弹窗：联系方式 sheet
- 已解锁：直接展示 contact
- 未解锁：调 `contact.unlock`，成功后同屏切换显示

---

### 全局事件（app.js）
| 事件 | 接口 |
| --- | --- |
| `onLaunch` | `user.login` |
| `onShow`（切换前台） | 可选 `user.login` 做 heartbeat，更新 `lastActiveAt` |
| `onError` | 上报到云函数（可选） |

### 分享
- `onShareAppMessage`：标题 "我在乾缘找到了一位 Ta，快来帮我把把关" + path 带上 `xyCode` 或 profile id
- 分享进入时 `onLoad options` 读取参数，自动打开对应 detail
