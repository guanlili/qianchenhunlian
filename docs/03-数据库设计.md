## 03 · 数据库设计

所有集合使用微信云数据库（JSON 文档型）。权限统一：**仅创建者可读写**，服务端用云函数的管理员权限访问。

### 命名约定
- 集合名使用小写下划线复数形式：`users`, `profiles`, `favorites`。
- 文档 id 使用云数据库自动生成的 `_id`。
- 业务主键（如寻缘号）单独字段存，不用作 `_id`。
- 时间统一用 server 时间戳 `db.serverDate()`，字段名 `createdAt` / `updatedAt`。

---

### 集合 1：`users` · 用户账号
一个微信用户 = 一条记录。负责登录、寻缘号、权益余额、通知订阅状态。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `_id` | string | ✓ | 自动 |
| `openid` | string | ✓ | 微信 openid（唯一索引） |
| `unionid` | string |  | 可选 |
| `xyCode` | string | ✓ | 寻缘号，8 位数字，生成规则见下 |
| `unlockBalance` | int | ✓ | 解锁次数余额，默认 3 |
| `notifyEnabled` | bool | ✓ | 是否订阅每日推荐 |
| `lastNotifyPermitAt` | date |  | 最近一次订阅消息 accept 时间（用于再次 confirm） |
| `followed` | bool |  | 是否已关注公众号（客户端上报） |
| `verified` | string |  | 实名认证状态 `none/pending/passed/rejected` |
| `status` | string | ✓ | `active / blocked` |
| `createdAt` | date | ✓ |  |
| `updatedAt` | date | ✓ |  |

**索引**：`openid` 唯一；`xyCode` 唯一。

**寻缘号生成**：8 位随机数字，查重后写入；碰撞重试 3 次，仍冲突则升到 9 位。

---

### 集合 2：`profiles` · 相亲资料（本人信息）
与 users 一对一，单独存以便做字段级更新和审核。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `_id` | string | ✓ |  |
| `openid` | string | ✓ | 外键 → users.openid（唯一索引） |
| `relation` | string | ✓ | 与相亲者关系：`父母/本人/朋友/亲戚` |
| `gender` | string | ✓ | `男/女` |
| `year` | int | ✓ | 出生年份 |
| `height` | int | ✓ | cm |
| `edu` | string | ✓ | 学历枚举 |
| `income` | string | ✓ | 年收入枚举 |
| `marriage` | string | ✓ | `未婚/离异未育/离异已育/其他` |
| `origin` | string |  | 户籍省市县 |
| `location` | string | ✓ | 居住省市县 |
| `hometown` | string |  | 家乡 |
| `job` | string |  | 职业 |
| `hasHouse` | string |  | `有婚房/无婚房/有购房计划` |
| `hasCar` | string |  | `有车/无车` |
| `bodyType` | string |  |  |
| `desc` | string |  | 相亲描述，最长 240 |
| `photos` | array<string> |  | 云存储 `fileID` 数组，首张为头像 |
| `tone` | string |  | UI 配色标签（前端生成，持久化可选） |
| `glyph` | string |  | UI 显示字（前端生成） |
| `contact` | object | ✓ | `{ wechat, phone }`，对外不直接下发 |
| `auditStatus` | string | ✓ | `pending / approved / rejected` |
| `auditReason` | string |  | 审核驳回原因 |
| `progress` | int | ✓ | 资料完善度 0-100，每次 update 由云函数重算 |
| `stats` | object | ✓ | `{ likes, hot, viewedCount }` |
| `lastActiveAt` | date |  | 上次打开小程序时间 |
| `createdAt` / `updatedAt` | date | ✓ |  |

**索引**：`openid` 唯一；`(gender, auditStatus, updatedAt)` 复合索引，供推荐查询用。

### 集合 3：`criteria` · 择偶要求
与 profiles 一对一。分开建是为了：①资料和择偶常常分开编辑；②后续可能做多套择偶模板。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `_id` | string |  |  |
| `openid` | string | ✓ | 唯一索引 |
| `year` | object |  | `{ min: 1987, max: 1992 }`，任一字段可空 |
| `height` | object |  | `{ min: 140, max: 156 }` |
| `income` | string |  | 单选枚举，`不限` 可为 null |
| `edu` | string |  | 单选枚举 |
| `marriage` | string |  |  |
| `house` | string |  |  |
| `origin` | array<string> |  | 多选省市县 |
| `location` | array<string> |  | 多选 |
| `note` | string |  | 补充说明 |
| `progress` | int | ✓ | 择偶要求完善度 |
| `createdAt` / `updatedAt` | date | ✓ |  |

### 集合 4：`favorites` · 收藏
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `_id` | string |  |  |
| `openid` | string | ✓ | 收藏发起人 |
| `targetOpenid` | string | ✓ | 被收藏者 |
| `createdAt` | date | ✓ |  |

**索引**：`(openid, createdAt DESC)` 用于"我收藏的"列表；`(openid, targetOpenid)` 唯一索引防重复。

---

### 集合 5：`unlocks` · 联系方式解锁记录
每次解锁写一条，用于展示 "已解锁"、扣减权益、防止重复扣费。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `_id` | string |  |  |
| `openid` | string | ✓ | 解锁发起人 |
| `targetOpenid` | string | ✓ | 被解锁资料 |
| `xyCode` | string | ✓ | 目标寻缘号（冗余便于展示） |
| `costBalance` | int | ✓ | 本次消耗次数，一般为 1 |
| `createdAt` | date | ✓ |  |

**索引**：`(openid, targetOpenid)` 唯一（再次"解锁"直接命中已有记录，不重复扣次数）；`(openid, createdAt DESC)` 查"我解锁过的"。

---

### 集合 6：`views` · 浏览记录（谁看过我）
详情页进入时写一条，聚合后作为 `users.stats.viewedCount` 与"看过我的人"列表数据源。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `_id` | string |  |  |
| `openid` | string | ✓ | 浏览人 |
| `targetOpenid` | string | ✓ | 被浏览者 |
| `createdAt` | date | ✓ |  |

**索引**：`(targetOpenid, createdAt DESC)`；`(openid, targetOpenid, createdAt)` 便于同一人 24h 内不重复计数（逻辑层控制）。

---

### 集合 7：`contact_intents` · 联系意向
列表页"联系对方"、详情页"发消息"写一条，供被联系者收到订阅消息。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `_id` | string |  |  |
| `fromOpenid` | string | ✓ |  |
| `toOpenid` | string | ✓ |  |
| `message` | string |  | 未来扩展 |
| `status` | string | ✓ | `sent / read / replied` |
| `createdAt` | date | ✓ |  |

---

### 集合 8：`daily_recommendations` · 每日推荐快照
定时任务每日凌晨为每个活跃用户生成一批推荐结果，前端直接读当天快照。避免每次请求都跑匹配计算。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `_id` | string |  |  |
| `openid` | string | ✓ | 接收推荐的用户 |
| `date` | string | ✓ | `YYYY-MM-DD`（北京时区） |
| `profileIds` | array<string> | ✓ | 推荐的 profile _id 列表，已排序 |
| `createdAt` | date | ✓ |  |

**索引**：`(openid, date)` 唯一。

---

### 集合 9：`notify_tokens` · 订阅消息模板配额
用户每次 `acceptSubscribeMessage` 后，记录一次可发送配额。发送后减 1。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `_id` | string |  |  |
| `openid` | string | ✓ |  |
| `templateId` | string | ✓ | 微信模板 ID |
| `remaining` | int | ✓ | 剩余可发送次数 |
| `updatedAt` | date | ✓ |  |

---

### 初始化脚本
后端在 `cloudfunctions/admin/` 里提供一次性 `initCollections` action，功能：
- 检查并创建以上 9 个集合
- 按需插入 3–5 条种子数据（从 `data.js` 搬过来）
- 创建索引（云开发提供 API 或手动在控制台建）

开发者在本地跑一次后把种子逻辑删掉，避免生产误调。
