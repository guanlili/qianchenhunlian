## 01 · 项目现状与目标

### 1. 当前进度
- **前端 UI（miniprogram/pages/index）**：8 个屏幕（home / detail / my-profile / me / profile-edit / criteria-edit / filter / quick-fill）+ 2 个弹窗（联系方式、option sheet）已经基本搭完，交互可跑通，但数据全部是本地 mock。
- **后端**：未开始。只有微信云开发 quickstart 模板里的 `sales` 示例集合和 `quickstartFunctions` 云函数，与业务无关。
- **配置**：`miniprogram/app.js` 的 `globalData.env` 为空，`wx.cloud.init` 已加空值保护（env 未填时跳过初始化）。
- **示例代码**：`miniprogram/pages/example/index`（云开发示例页）和 `cloudfunctions/quickstartFunctions`（示例云函数）未清理，后续应移除或替换。

### 2. 主要缺口
| 模块 | 缺口 |
| --- | --- |
| 登录与用户体系 | 没有登录流程；`寻缘号 53652255` 是硬编码 |
| 资料管理 | 资料录入只存本地 data，不落库；无图片上传 |
| 择偶要求 | 同上 |
| 每日推荐 | 列表来自本地，无推荐算法、无"今日/看看新人/错过的人"真实数据源 |
| 筛选 | 选项在前端，筛选结果没有真实数据来源 |
| 详情页 | 数据 mock；点赞数/人气值没有持久化 |
| 收藏 | 点击只改前端 state |
| 解锁联系方式 | 无权益体系、无扣减逻辑、没有真实联系方式字段 |
| 消息/联系意向 | 仅 toast 提示 |
| 每日推荐通知 | 没接 `wx.requestSubscribeMessage` |
| 实名认证 | 纯占位 |
| 审核机制 | 资料"审核中"只是个文案 |
| 分享 | 没接 `onShareAppMessage` |
| 统计（点赞/人气值/看过我） | 写死 |

### 3. MVP 目标（第一个可上线版本的范围）
必须完成：
1. 微信 openid 静默登录 + 寻缘号注册
2. 我的资料 / 择偶要求 读写与落库
3. 资料图片上传到云存储
4. 资料完善度真实计算
5. 每日推荐列表（简单推荐：按择偶条件倒序 + 新资料优先）
6. 详情页展示真实数据
7. 收藏 / 取消收藏 持久化
8. 联系方式解锁（用"解锁次数"这类简单权益，数量先做后台赠送，不接支付）
9. "看过我" 浏览记录
10. 公众号订阅消息接入（每日推荐通知）
11. 分享到好友 / 群
12. 资料审核状态位（先做字段，人工审核或自动通过可切换）

### 4. 非目标（一期不做）
- IM 实时聊天（用系统订阅消息 + 外部联系方式替代）
- 付费 / 订单 / 发票
- 实名认证对接真实第三方（先留占位）
- 音视频
- 复杂推荐算法（协同过滤等）
- 管理端后台（先用云开发控制台直接管数据）

### 5. 成功判定
一期上线后：
- 新用户可完成 登录 → 填资料 → 浏览推荐 → 收藏 → 解锁联系方式 全流程
- 所有数据真实落库，刷新小程序不丢
- 订阅消息能正常发送
- 日活 10 人跑通不出错
